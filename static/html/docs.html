CloudKarafka Management HTTP API

Introduction

Apart from this help page, all URIs will serve only resources of type application/json, and will require HTTP basic authentication (using SASL SCRAM-SHA-512 stored in ZooKeeper).

For all PUT/POST/DELETE actions the user need to have the cluster write ACL rule.

Routes

GET PUT DELETE POST Path Description
* /api/brokers List of online broker ids.
* /api/brokers/{id} Detaild info about the broker.
* /api/brokers/{id}/metrics Runtime metrics gathered via JMX for given broker id.
* * /api/topics List of topics.
* * * /api/topics/{name} Detais about given topic.
* /api/topics/{name}/metrics Runtime metrics gathered via JMX for given topic.
* * /api/topics/{name}/config Configs for given topic.
* /api/topics/{name}/reassigning Return status of reassigning partitions if there is any.
* /api/topics/{name}/{partition} Detais about given topic partition.
* /api/topics/{name}/{partition}/metrics Runtime metrics gathered from JMX for given topic partition.
* /api/consumers List of active consumers.
* /api/consumers/{name} Details about given consumer.
* /api/whoami List of permissions for authenticated user.
* * /api/users List users.
* * /api/users/{name} Details for given user.
* /api/acls List of all Acl rules.
* * /api/acls/{type}/{resource}/{username} Get specific Acl rule.
